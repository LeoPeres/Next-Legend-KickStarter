  Use the destructive variant for potentially dangerous actions - Use the outline or ghost variants
  for secondary actions - Use the link variant for navigation actions that should appear as links -
  Choose appropriate sizes based on the context and importance of the action

## Accessibility Considerations

The Button component is designed with accessibility in mind:

- It maintains a proper contrast ratio for all variants
- It includes focus states for keyboard navigation
- It uses appropriate ARIA attributes
- It supports keyboard interaction

## Design Tokens

The Button component uses the following design tokens:

- `--primary`: Background color for the default variant
- `--primary-foreground`: Text color for the default variant
- `--destructive`: Background color for the destructive variant
- `--destructive-foreground`: Text color for the destructive variant
- `--secondary`: Background color for the secondary variant
- `--secondary-foreground`: Text color for the secondary variant
- `--accent`: Background color for hover states in outline and ghost variants
- `--accent-foreground`: Text color for hover states in outline and ghost variants
- `--ring`: Color for the focus ring
